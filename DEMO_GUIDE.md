# Qdrive 프로토타입 — 시연 가이드 (전 기능 인벤토리)

> 라이브: https://hyegeungim3-git.github.io/qdrive-proto/
> 이 문서는 **시연자가 화면을 보며 따라가는 대본 + 놓치기 쉬운 숨은 기능 체크리스트**입니다.
> 🔴 = 반드시 보여줄 킬러 기능 · 🔍 = 클릭·조작해야 드러나는 숨은 기능

---

## 0. 시연 전 세팅 (30초)

1. 라이브 링크 열기 (또는 발표 노트북에서 `npm run dev` — 실차 데이터까지 쓰려면 로컬)
2. 헤더 배속을 **5×** 로 (데이터가 적당히 쌓이는 속도). 데이터를 미리 쌓으려면 60×로 10초 돌린 뒤 5×로
3. 상단 **"시연"** 점선 박스 = 발표 리모컨 (급감속·고장·민원·배차·사고·날씨). 어느 탭에서 눌러도 **토스트 + "○○탭으로 이동"** 버튼이 떠서 효과가 어디서 보이는지 안내됨
4. 🔍 우측 끝 **☀️/🌙 테마 토글** — 밝은 회의실=라이트, 스크린 투사=다크
5. 🔴 우하단 **✨ AI 코파일럿 버튼** — 어느 탭에서든 떠 있는 자연어 관제 비서 (아래 "AI 3층 구조" 참고)

---

## ✨ AI가 3개 층으로 들어간다 (시연 관통 메시지)

Qdrive의 AI는 화면 곳곳에 흩어진 게 아니라 **3개 층**으로 정리돼 있습니다. 시연 중 이 프레임을 한 번 말해주면 모든 AI 기능이 하나로 꿰집니다.

| 층 | 성격 | 어디서 | 한 줄 |
|---|---|---|---|
| **조회형** | 물어보면 답 | 🔴 **✨ AI 코파일럿**(우하단 상시) | "고장 위험 차량?" → 실데이터로 답 + 근거 + [조치 실행]·[화면 이동] |
| **실행형** | 승인하면 처리 | 🔴 **AI 업무센터** 탭 (§5) | 문서·공문 초안까지 생성 → 담당자 [승인] → 실행 |
| **개인형** | 각자 자기 일 | 🔴 **에이전트 플랫폼** 탭 (§6) | 회사·기사가 각자 쓰는 AI, 신청↔승인으로 연결 |

- 화면 안 곳곳의 **Agentic 자동 조치**(배차 권고·시민안내 배너·차고지 재배정)는 이 3층이 이미 스며든 형태.
- 공통 원칙: **AI는 제안·초안, 결정·확정은 사람** (평가·처벌·정산 확정은 자동화하지 않음).

## 1. 시티 대시보드 (대구시 관점) 🔴

가장 정보 밀도가 높은 화면. 좌·중·우 3단.

**좌측 컬럼**
- 🔴 **📑 AI 정책 보고서** (맨 위 보라 버튼) — 전 데이터 총괄 자동 보고서. **열면 전체 화면 오버레이**: 운행·안전·재정·시민 4문단 + 정책 제언, 문단마다 **근거 수치 칩**. 🔍 헤더에 **기간 셀렉터(오늘/1주/1개월/1년)** + 📋복사
- 🔍 **⚙ 위젯 구성** — 8개 위젯 on/off 체크박스, **브라우저에 저장됨** (발표 구성 커스터마이즈)
- **운행 현황** 도넛(운행률) · **이용객 수**(실시간 승차 집계, 교통카드/현금 분해) · **🚨 돌발정보**(사고/고장/공사/기타 발생·처리중) · **이상 현황**(유형별 최근5분/누적) · **📈 혼잡 추이**(🔍 우측 "전일 비교 ON/OFF" 토글)

**중앙 지도**
- 🔴 **실제 버스 아이콘**(노선색 차체+창문+바퀴, 진행방향 반전), 위험운전 시 붉은 글로우
- 🔴 **🔥 위험운전 히트맵** 토글(우상단)
- 🔴 **📡 대구 BIS 실데이터 → "연동 시작"** — 실제 대구 버스(급행1·3·10·순환2)가 **하늘색 아웃라인 버스**로 지도에 뜸 (15초 갱신, 시뮬 버스와 나란히). 🔍 뜨고 나면 우상단에 **노선 필터 칩**(급행1 18…) 개별 on/off
- 🔍 좌상단 **날씨·시각 칩**, 좌하단 노선 범례
- 🔍 **돌발정보 항목 클릭 → 지도가 그 지점으로 날아가고 보라 포커스 링** 표시

**우측 컬럼**
- KPI 4종 · BIS 패널 · **노선 평가·준공영제 정산 검증**(BMS×DTG 교차, 🔍 회차 쌓이면 "인가노선 87% 운행" 정산 플래그 노출) · **위험운전 실시간 피드**(🔍 정당 판정 이벤트는 🛡 초록 배지)

## 2. 운수사 관제 (버스회사 관점) — 서브탭 6종 🔴

상단에 서브탭 **7개**. **여기가 기능이 가장 많이 숨어있는 곳.**

### ① 관제 현황
- 🔴 **🤖 AI 추천 조치** — "🚌 배차" 리모컨 누르면 여기 **배차간격 권고**가 뜸 → **[승인]** 클릭 → 다음 정류장에서 실행·검증 (예측→조치안→승인→실행→검증 루프)
- 🔴 **🎙 기사 상황 설명** — 기사가 남긴 설명 목록 → **[설명 확인]** 클릭 시 감점 즉시 복원 (아래 5번 상황 설명과 연결)
- 🔴 **고장예측 배너** — "🔧 고장" 누르고 60×로 돌리면 냉각수온 곡선이 임계치 도달 → 예측 알림 + **정비 작업지시 [발행]**
- 차량·기사 랭킹 테이블 · 정비비 예측 · eTAS 자동제출
- 🔍 테이블 **배차간격 컬럼** — 각 차량의 앞차 간격(선두/몰림⚠/정상), 셀에 마우스 올리면 앞·뒤차 상세 (기사 앱 앞뒤차 카드와 같은 데이터)

### ② 운행 이력 🔴🔍
- 차량 선택 → 회차별 운행 시작·종료·거리·연비·CO₂ 테이블 + 합계
- 🔴 **안전운전 리포트** — 위험운전 8종 **버튼**. 발생한 유형만 활성
- 🔴🔍 **유형 클릭 → 발생 내역 목록 → 행 클릭 → 바로 아래 지도가 아코디언으로 펼쳐짐** (그 지점 포커스 링·라벨 + 시각·정류장·속도·RPM·정당/감점 판정). **탭 이동 없이 관제 안에서 지도 확인** ← 최근 추가된 핵심

### ③ AI 리포트 🔴🔍
- 🔴 **자동 생성 운영 리포트** — 운행 총평/안전/코칭 제안/차량 상태(+기상) 문단, 문단마다 근거 칩
- 🔍 헤더 셀렉터 **①기간(오늘/1주/1개월/1년)** — 바꾸면 볼륨 수치 전부 스케일 + 각주에 "확장 배수·실증 대체" 명시
- 🔴🔍 헤더 셀렉터 **②대상(전체 ↔ 기사 9명 각각)** — **운전원별 리포트**(종합 진단·위험행태·맞춤 개선 제안, 사내 순위·밀도 비교). 광주 "운전원별 리포트" 재현

### ④ 연료·에코 AI 🔴🔍 ← 최근 추가, 연료절감 차별화 핵심
- 🔴 **연료 낭비 원인 분해 waterfall** — 낭비를 4요인(운전습관·급조작·공회전·냉방)으로 쪼개고 **코칭 가능분 vs 환경 요인** 구분 (급조작·습관은 코칭으로 줄임, 냉방은 환경)
- 🔴 **AI 절감 시뮬레이션** — 코칭 가능 낭비를 **대구 CNG 1,513대 연간으로 환산(약 X억원)** — "증액 없이 재정 아끼는 사업" 근거
- 🔴🔍 **차량별 경제운전(관성주행) 순위** — 정류장 전 발 떼기 비율 기준, 모범 기사가 상위·연비와 낭비 1위 요인 병기
- 💡 강조 멘트: "테너지는 급가속을 **센 뒤 벌점**을 줬습니다. 우리는 낭비가 일어나기 **전에** 관성주행을 안내합니다 — 예방형 연료절감"

### ⑤ 진단 스캐너 🔍
- 🔴 **OBD/CAN 1초 센서값 라이브 테이블** 21종(ADC·ECU·GPS·SCR), 추이 미니바
- 🔍 차량 셀렉터로 차량 전환 · **고장 시나리오 진행 중이면 냉각수온 행이 붉게 상승**
- 🔍 하단 설명: **GPS 속도 vs 내부 차량속도 차이** — Qdrive가 GPS 아닌 차량속도를 쓰는 이유(테너지 실패 요인)

### ⑥ AI+ 정비도우미 🔍
- 🔴 챗 UI — 차량번호 입력 시 최근 진단 요약, 🔍 **추천 질문 칩**(P0299 고장코드·냉각수온 원인·브레이크 점검) 클릭 응답
- 하단: "실서비스는 LLM+RAG, 데모는 스크립트" 명시(정직성)

### ⑦ 차고지·충전 🔍

## 3. 기사 앱 (운전자 관점) — 12.3" 태블릿 🔴

가로형 태블릿 프레임. 🔍 좁은 화면에선 자동 축소(빔프로젝터 대응).

- 🔴 **"⚡ 급감속" 리모컨 → 상단에 빨강 경고 토스트 + [🎙 상황 설명하기]** 버튼
  - 🔴🔍 **설명 버튼 → 음성 인식(한국어)** 켜짐. 말하면 텍스트로 기록(미지원 환경은 버튼 폴백) → 운수사 상황 설명함으로
- 🔴🔍 **방어운전 자동 인정** — "🚨 사고" 낸 뒤 그 근처에서 "⚡ 급감속"을 내면, 빨강 경고 대신 **초록 "🛡 회피 기동 인정 — 감점 없음" 배너** (돌발 반경/폭우/정류장/군집 4개 맥락 자동 판정). 랭킹 카드에 **🛡 방어 크레딧** 칩

- 🔴🔍 **🌿 예측형 에코 코칭** — 정류장 접근(약 160m 전)이나 앞차 근접을 감지하면 **"지금 가속 페달을 떼고 관성으로 진입하면 연료가 절약됩니다"**를 미리 안내 (낭비 발생 전). 정속 주행 중엔 "연비 최적 구간, 좋아요!" — 상황에 따라 문구가 실시간으로 바뀜
- 🔴 **🌿 경제운전 점수 게이지** + 오늘 연료 낭비 1위 요인
- 🔴🔍 **🔗 앞·뒤차 배차 간격 카드** — 뒤차/이상/앞차 3분할(분), 진행 바에 앞·뒤차 위치도 반투명 버스로 표시. **몰림(앞차 근접) 시 앰버 강조** + "정류장에서 여유를 두면 배차 고르게" 코칭 (관제 배차 권고와 같은 데이터)
- 사내 안전운전 순위(🛡 방어 크레딧 칩) · 휴게·교대 · CO₂ 절감
- 🔍 **날씨 전환 시** 상단에 "폭우 노면 주의" 등 **운행 지침 배너** 자동 표시
- 🔍 **하차 예약이 걸리면** 헤더에 "🔔 ○○ 하차 예약 1명" 사전 표시(무정차 예방)

## ✨ AI 코파일럿 (조회형 · 어느 탭에서든) 🔴

우하단 **✨ 버튼**을 누르면 뜨는 플로팅 챗. **탭을 옮겨 다니며 눈으로 찾던 걸 한 줄로 묻는다.**

- 🔴 **자연어 질문 → 실시간 스냅샷 조회 답변** — "지금 위험 차량 있어?", "고장 위험 차량?", "배차 몰린 노선?", "연료 절감 얼마?", "민원 몇 건?"
- 🔴🔍 **답에 근거 칩 + 액션 버튼** — 예: "배차 권고 생성" 버튼 → 실제 운수사 관제에 배차 권고가 생김 → **[운수사로 이동]** 링크로 바로 확인 (조회가 실행·이동으로 이어짐)
- 🔍 하단 **추천 질문 칩** — 처음 여는 사람도 무엇을 물을지 바로 앎
- 하단 각주: "데모=규칙 기반 매칭, 실서비스=LLM+함수 호출" (정직성)

## 5. AI 업무 자동화 센터 (실행형) 🔴 ← 신규

**버스회사·대구시가 반복하는 행정 업무를 AI가 초안까지 만들고, 담당자는 승인만.** 상단 **소유자 토글**(🏢 버스회사 / 🏛 대구시)로 6개 업무 카드 전환.

- 🔴 **버스회사 3종** — ①민원 회신문 초안 ②정비 작업지시서 ③코칭 대상 선정·안내문
- 🔴 **대구시 3종** — ④준공영제 정산 공문 ⑤시의회 답변 자료 ⑥탄소 실적 보고
- 🔴🔍 **카드 펼치면 실제 문서가 생성됨** — 스냅샷 실데이터로 채운 **공문·회신문 전문(全文) 미리보기**(`<pre>` 블록). 제목·수신·본문·수치까지 완성형
- 🔴 **자율성 배지 L2(추천)/L3(승인 후 실행)** — L3 카드의 **[승인·실행]** 누르면 실제 상태 전이(민원 답변 단계 전진·작업지시 발행 등)
- 준비 안 된 업무는 "대기" 상태로 흐릿하게 — 데이터가 쌓이면 활성 (예: 민원이 있어야 회신문 생성)
- 하단 각주: 실서비스는 LLM 문서 생성, **최종 발송·정산 확정은 담당자 판단**

---

## 6. 회사·기사 에이전트 플랫폼 (개인형) 🔴 ← 신규

**운수회사 안에서 회사(관리자)와 기사가 각자 쓰는 개인 AI.** 둘이 **신청↔승인**으로 연결되는 게 핵심. 상단 **역할 토글**(🏢 회사(관리자) / 🧑‍✈️ 기사(김성호)).

- 🔴 **역할별 "오늘의 브리핑"** — 회사: 운행 대수·정비·승인 대기·평균 점수 / 기사: 오늘 배차·안전점수·사내 순위·다음 교대
- 🔴 **역할별 자연어 에이전트** — 회사: "결원 있어?"·"정비 입고 차량?"·"코칭 대상 기사?" / 기사: "오늘 내 배차?"·"내 안전점수?"·"내 차 점검 있어?"·"이번 달 급여?" (답에 근거 칩)
- 🔴🔍 **크로스롤 워크플로우** — 기사 화면 빠른 버튼 **[🏖️ 휴가 신청]·[🎙 상황 설명 제출]·[🎓 교육 일정]** → 회사 역할의 **📥 기사 요청 승인함**에 즉시 뜸(탭 배지 숫자) → 회사가 **[승인]/[반려]** → 기사 화면 **📤 내 신청 현황**에 "✓ 승인됨" 반영
- 🔍 회사 역할 토글에 **승인 대기 건수 배지** — 처리할 일이 있으면 숫자로 알림
- 하단 각주: 실서비스는 역할별 LLM 에이전트+도구 호출(조회·신청·승인·정산), **급여·평가 확정은 담당자 최종 판단**

## ⭐ 킬러 시연 동선 (12분) — "데이터가 4주체를 흐른다"

숨은 기능을 자연스럽게 다 보여주는 순서:

1. **승객 앱** → 🔔 하차 예약(자동) 걸고 → **📢 불편사항 접수** *(승객의 목소리가 데이터로)*
2. **시티 대시보드** → 민원 카드 **"데이터로 원인 확인"** → 증빙 자동매칭(GPS·DTG·DVR 타임라인·AI점수) → 다음 단계서 **답변 초안** *(감이 아니라 증빙)*
3. **"⚡ 급감속"** → **기사 앱** → 경고 → **🎙 상황 설명(음성)** → **운수사 상황 설명 [확인]** → 감점 복원 *(감시 아닌 코칭)*
4. **"🚨 사고"** 낸 뒤 근처에서 **"⚡ 급감속"** → 기사 앱에 **🛡 회피 기동 인정** *(사고 막은 기사는 점수를 준다)*
5. **"🚌 배차"** → 운수사 **[승인]** → 실행·검증 / **"🔧 고장"**+60× → 예측 배너 → **진단 스캐너**서 냉각수온 붉게 → **작업지시 발행** *(AI는 조치안, 결정은 사람)*
6. **"🌦 날씨"**(폭우) → 시티 예측·기사 지침·승객 안내·차고지 예비차 **4화면 동시 반영**
7. **기사 앱**에서 **🌿 예측형 에코 코칭**("정류장 앞 관성주행") + **앞·뒤차 배차 간격** 보여주고 → **운수사 → 연료·에코 AI**: 낭비 원인 분해 + **연간 억원 절감 시뮬레이션** *(테너지는 사후 벌점, 우리는 예방 — 증액 없이 재정 절감)*
8. **운수사 → 운행 이력** → 위험운전 유형 클릭 → 이벤트 행 클릭 → **아코디언 지도로 발생지 확인**
9. **운수사 → AI 리포트**(전체→기사별, 기간 1개월) / **시티 → 📑 정책 보고서**(1년) *(같은 데이터, 직급별 언어)*
10. 🆕 **✨ AI 코파일럿**(우하단) 열고 "고장 위험 차량?" → 답+근거 → **"배차 권고 생성"** 버튼 → [운수사로 이동] *(조회형 AI가 실행·이동으로)*
11. 🆕 **AI 업무센터** → 민원 회신문 카드 펼쳐 **공문 전문 생성** → **[승인·실행]** → 시티 민원 단계 전진 *(실행형: 초안은 AI, 발송은 사람)*
12. 🆕 **에이전트 플랫폼** → 기사 역할서 **[휴가 신청]** → 회사 역할 **승인함**에 뜸 → **[승인]** → 기사 "✓ 승인됨" *(개인형: 회사·기사가 한 플랫폼에서 연결)*
13. **BIS 연동 시작** → 실제 대구 버스가 지도에 *(협약 없이도 실물)* → **실증 리포트·로드맵**으로 마무리

> 💡 10~12는 **"AI 3층 구조"**(조회형·실행형·개인형)를 순서대로 시연하는 구간. 시간이 빠듯하면 이 3개 중 하나만 골라 보여줘도 메시지는 전달됨.
