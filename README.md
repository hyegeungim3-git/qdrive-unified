# Qdrive 대통합 — 대구 시내버스 탄소중립 운영 플랫폼

두 선행 프로토타입을 **하나의 통일된 플랫폼**으로 통합한 정본. 사업 제안용 라이브 데모.

- **base** = `qdrive-proto` (Vite/React 19/TS/Tailwind 4) — 실 시뮬레이터 엔진 + 이해관계자 탭
- **이식** = `AI 기반 탄소중립 운영 플랫폼`(정적 HTML) — 탄소중립 서사·시뮬레이터·시민 페이지·경영 손익·라이브 코파일럿

> 통합 원칙: **살아있는 코어는 proto, 사업가치 레이어는 탄소.** 탄소 플랫폼의 서사·콘텐츠·What-if 시뮬레이터를 proto의 시뮬레이터 엔진에 연결해, 정적 목업을 실동작으로 승격시켰다.

## 실행

```bash
npm install
npm run dev     # http://localhost:5173
npm run build   # 프로덕션 빌드 (dist/)
```

> **빌드 함정**: `node_modules/.vite` 캐시가 손상되면 vite 8(rolldown)이 청크 렌더링 단계에서 네이티브 크래시(exit -1073740791)한다. `node_modules/.vite`와 `dist`를 지우고 재빌드하면 해결. 코드 문제 아님.

인터넷 필요 항목: 지도 타일(CARTO), (선택) BIS 실데이터·라이브 코파일럿. 나머지는 전부 로컬 동작.

## 구조 — 실단말 연동 시 교체 지점

```
src/
  sim/        ← 시뮬레이터 엔진 (types.ts=409/521 스키마·PacketSource / engine.ts / store.ts)
  views/      ← 이해관계자 탭 + CarbonAnalysis(탄소중립 분석) + CitizenPublic(시민 공개)
  components/ ← MapView · DemoControls · Copilot(AI 코파일럿) · ui
  worker/     ← Cloudflare BIS 프록시 (TAGO 전용)
```

## 원본과의 관계

이 저장소는 원본 두 프로젝트를 **복사**해 만든 **독립 새 git 저장소**(로컬)다. 원본(`qdrive-proto`, `qdrive-carbon-platform`)과 그 배포는 이 작업의 영향을 받지 않는다. 배포 시 새 저장소로 올리고 `vite.config.ts`의 `base`를 저장소명과 일치시킬 것.

## 주의 (데모용 목업 수치)

- 월/연 서사 수치·리포트는 일관성 있는 근사치이며 실측이 아님(화면에 명시). 일간 성과·안전 게이지·기사 순위 등은 시뮬레이터 엔진 실집계.
- 탄소 정합: 경유 배출계수 2.68 kgCO₂/L, 소나무 6.6 kgCO₂/그루·년, 경유 1,550원/L.
- 라이브 코파일럿·BIS 키는 브라우저 localStorage 전용 — **저장소 커밋 금지**.
