# Qdrive 프로토타입 — 대구 시내버스 통합 플랫폼 실증 데모

7월 사업 제안용 라이브 데모. **대구시·버스회사·운전자·승객 4개 이해관계자를 총괄**하는 플랫폼으로, 실차·실단말 없이 **공단 SDK 패킷(409 위험운전 / 521 운행기록) 스키마 그대로** 시뮬레이션하여, 실단말 연동 시 데이터 소스만 교체하면 제품으로 이어지는 구조.

참고 소스: 광주 시내버스 안전 모니터링 완료보고(차량 진단·정비 축) + 노션 「대구시 시내버스 통합」 아이데이션(Agentic AI — 승인 기반 실행, 민원 증빙 자동매칭, 배차간격 권고) + 기존 Qdrive 자산(eTAS·탄소중립·기사 앱).

## 실행

```bash
npm install
npm run dev   # http://localhost:5173
```

인터넷 필요 항목: 지도 타일(CARTO)뿐. 나머지는 전부 로컬에서 동작.

## 📡 대구 BIS 실데이터 연동 (선택)

시티 대시보드의 "대구 BIS 실데이터" 패널에서 **실제 대구 시내버스**(급행1·급행3·순환2) 위치를 TAGO 오픈API로 받아 시뮬레이션 버스와 나란히 지도에 표시할 수 있다 (속이 빈 하늘색 마커 = 실차, 15초 갱신).

1. [공공데이터포털](https://www.data.go.kr) 회원가입 → [국토교통부_(TAGO)_버스위치정보](https://www.data.go.kr/data/15098533/openapi.do)와 [버스노선정보](https://www.data.go.kr/data/15098529/openapi.do) **활용신청** (자동승인, 즉시 발급)
2. 마이페이지의 **일반 인증키**를 패널에 입력 → 연동 시작
3. CORS는 Vite 개발서버 프록시(`/tago`)가 중계하므로 `npm run dev` 환경에서 동작 (운영 배포 시 별도 프록시 필요)

※ 심야 등 버스 미운행 시간에는 데이터가 없을 수 있음. 전략적 의미: BIS 위치(대구시 공개 자산) × 차량내부 데이터(Qdrive) 결합을 협약 없이 시연 — "BMS를 대체가 아니라 완성한다"는 메시지의 증거 장면.

## 데모 시나리오 (12분 — "4주체 순환" 스토리)

헤더 우측 컨트롤 바: 배속(1×~60×) · 일시정지 · **⚡ 급감속** · **🔧 고장 시나리오** · **📢 민원 접수** · **🚌 배차 권고**

1. **승객 앱** — 반월당 정류장: 도착 신뢰도·혼잡도·휠체어 공간. **"불편사항 접수하기"** 클릭 → *"승객의 목소리가 데이터 파이프라인에 들어간다"*
2. **시티 대시보드** — 민원 카드에서 **"데이터로 원인 확인"** → 조사 에이전트가 GPS 궤적·차량속도·DTG 409·문 개폐·DVR 클립을 자동매칭, 사실 가능성 점수 + 차량·기사 식별 → 다음 단계에서 **답변 초안** 생성 → *"민원 처리가 감이 아니라 증빙이다"*
3. **기사 앱** — **⚡ 급감속** 클릭 → 실시간 경고 팝업 + 보정 점수 + 휴게 알림 → *"감시가 아니라 코칭"*
4. **운수사 관제** — **🚌 배차 권고** 클릭 → "70초 대기 → 배차간격 완화" 권고 카드 → **승인** → 다음 정류장에서 실행완료·검증 → *"AI는 조치안, 결정은 사람"* (Agentic 원칙)
5. **🔧 고장 시나리오** → 냉각수온 곡선 → 예측 배너 + **작업지시 초안** → **발행** → **진단 스캐너** 서브탭에서 냉각수 온도 행이 붉게 상승하는 것 확인 → **AI+ 정비도우미**에 차량번호 입력 → 진단 요약·권장 점검 순서 → *"운행 중단 전에 예방 정비, 정비사에게는 AI 도우미"*
6. **🌧 날씨 버튼** (폭우 전환) → 시티: 수요·지연·사고위험 예측 카드 / 기사 태블릿: "제동거리 1.5배" 지침 / 승객 앱: 호우 지연 안내 / 차고지: 예비차 선배정 → *"외부 변수 하나가 4개 화면에 동시에 반영된다"*
7. 다시 **승객 앱** — 시민안내 에이전트가 "예방 정비로 배차 조정" 안내문 자동 표출 + 내 민원 처리 현황 → *"운영 데이터가 시민 언어로 돌아온다 — 4주체 순환 완성"*
8. **실증 리포트** → 연비 +8%대·위험운전 −57% → **로드맵** → 4-이해관계자 커버리지와 1·2·3차 확장으로 마무리

## 구조 — 실단말 연동 시 교체 지점

```
src/
  sim/
    types.ts    ← 공단 패킷 스키마 (409/521) + PacketSource 인터페이스
    engine.ts   ← 시뮬레이터 (주행·페르소나·연비·고장) ★ 실단말 연동 시 교체 대상
    routes.ts   ← 대구 노선 근사 폴리라인 ★ BIS 노선 GIS로 교체 대상
    store.ts    ← 엔진 싱글턴 (여기서 SimEngine → RealPacketSource 스왑)
    geo.ts      ← 폴리라인 주행 계산
  views/        ← 시티/운수사/기사/리포트/로드맵 — 데이터 소스 교체와 무관하게 유지
  components/   ← 지도, 프리젠터 컨트롤, 공용 UI
```

## 주의 (데모용 목업 수치)

- 연비·CO₂ 계수(0.52 m³/km, 2.2 kg/m³)와 리포트 수치는 **일관성 있는 근사치**이며 실측이 아님 — 리포트 화면 하단에 명시됨
- 배출권 수익은 방법론 검토단계로 보수적으로 표기 (웹리서치 정정 반영)
